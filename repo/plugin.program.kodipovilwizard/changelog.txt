[COLOR yellow][B]v0.1.7[/B][/COLOR]
* Python 3.14 compatibility: resources/libs/common/logging.py
  used to do `try: from urllib.request import FancyURLopener
  except ImportError: from urllib import urlencode` -- both
  paths break on Python 3.14 (FancyURLopener was removed,
  `urllib.urlencode` was Py2-only). On Arch Linux + Py3.14 the
  whole wizard refused to start (clear.py -> logging.py import
  chain crashed at module load). Rewrote the failing import to
  a single `from urllib.parse import urlencode`, and replaced
  the `LogURLopener(FancyURLopener)` class + `post_log()` body
  with a `requests.post()` call (requests is already a wizard
  dependency). post_log behaviour is identical; the only
  visible change is that Linux + Py3.14 users can now actually
  launch the wizard.

[COLOR yellow][B]v0.1.4[/B][/COLOR]
* Critical hotfix on top of v0.1.3: the auto-set step also populates
  'buildversion', 'latestversion' and 'nextbuildcheck'. Without these
  the next check_build_update saw buildversion='' and treated any
  published version as newer, popping the Fresh-Install dialog that
  overwrites the user's userdata (wiping connected services). Also
  pushes nextbuildcheck 30 days out as a safety net.
[COLOR yellow][B]v0.1.3[/B][/COLOR]
* Auto-set 'buildname' setting on startup if empty and plugin.video.pov
  is present on disk. Fixes the silent breakage of auto_quick_update for
  users who installed via APK (where the wizard's Fresh Install flow
  never ran, so 'buildname' stayed empty and every quick_update gate
  no-op'd). With this in place, future quickfix updates and build
  update checks fire automatically on APK installs too.
